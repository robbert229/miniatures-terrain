# miniatures-terrain

A collection of parametric OpenSCAD models for tabletop miniature terrain.

The goal of this repository is to provide printable, customizable terrain pieces that are generated procedurally using OpenSCAD. Rather than storing dozens of static STL files, each project exposes parameters that allow terrain to be regenerated with different dimensions, layouts, and randomized variations.

## Repository Structure

```
miniatures-terrain/
├── README.md
├── sugarcane-field/
│   ├── README.md
│   ├── sugarcane-field.scad
│   ├── Makefile
│   └── ...
└── ...
```

Each subdirectory represents an independent terrain project.

## Requirements

- OpenSCAD
- GNU Make
- Bash (recommended)
- Optional:
  - Python 3
  - NumPy
  - trimesh
  - numpy-stl

## Building

Most projects include a Makefile.

Example:

```bash
make
```

Some projects generate multiple randomized variants by overriding OpenSCAD variables from the command line:

```bash
openscad \
    -D "seed=5" \
    -o output.stl \
    terrain.scad
```

or

```bash
make COUNT=25
```

depending on the project.

## Design Philosophy

Projects in this repository generally follow a few principles:

- Fully parametric whenever practical
- Printable on common FDM printers
- No supports required whenever possible
- Randomized layouts using deterministic seeds
- Optimized for tabletop gaming

Randomization is deterministic, allowing identical terrain to be regenerated by reusing the same seed.

## Printing

Most terrain is intended for:

- 0.4 mm nozzle
- 0.20 mm layer height

Although many models also print well using larger nozzles for faster production.
